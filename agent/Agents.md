# agent.md

## 权威关系树
- `root_source.md`：直接记录你的原始想法，唯一最高权威。
- `root_design.md`：Agent 根据 `root_source.md` 生成的总体设计。必须完全服从 `root_source.md`。
- 子组件设计（如 `module_*.md`）：必须服从 `root_design.md` 及上级文档。

## 开发流程
1. 读取 `root_source.md`，理解核心需求。
2. 生成 `root_design.md`（架构、模块划分、技术选型），不可偏离原始想法。
3. 逐级生成子组件设计，每级向上对齐。
4. 编码实现：严格按设计文档执行，变更需先更新文档并保持上级约束。

## 关键规则
- **上级支配下级**：下级文档不得与任何上级矛盾。
- **想法优先**：`root_source.md` 是一切源头，不可擅自修改。
- **Token 高效**：输出仅必要内容，避免解释、重复和格式冗余。
